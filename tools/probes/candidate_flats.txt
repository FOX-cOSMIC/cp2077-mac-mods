#
# HOW EACH LINE WAS CHOSEN
#   record   : confirmed present via `strings -8 tweakdb.bin | grep -xF` (all 41 records verified)
#   property : a SCALAR (Int32/Float/Bool) flat from the matching record-TYPE
#              block in reference/windows-tweakxl/data/ExtraFlats.yaml
#              (P1.10a WriteFlat only handles Int32/Float/Bool).
#   The record<->type binding (e.g. "Preset_* is WeaponItem") is INFERRED from
#   naming; that inference is exactly what the probe confirms/refutes.
#
# CONFIDENCE: no line is "high" until the probe HITs (cinema lesson). Tags below:
#   [M]  medium  — property strongly matches record type, likely universal on it
#   [L]  low     — plausible but type-binding or per-record presence uncertain
#
# FORMAT: one `Record.property` per line. `#` lines and inline `# ...` comments
# are ignored by the probe. ~100 candidates.
#
# NOTE on missing diversity buckets (honest, per F-020 + ExtraFlats reality):
#   - VEHICLE scalars: ExtraFlats `Vehicle` block is ALL String; only 7 Vehicle.*
#     records exist (tuning helpers). No scalar vehicle flats are derivable. The

#     VehicleDataPackage.exiting) have no confirmable record names in the pool.

#   - ECONOMY/price: `price` is TweakDBID (foreignType ConstantStatModifier), NOT

#   - MOVEMENT speed / FOV: driven by the StatModifier system, not scalar flats.
#   Buckets reallocated to where real scalars exist (weapon/grenade/projectile/
#   character/status-effect).

# ============================================================================
# Probe interface for Schema (next turn)
# ============================================================================
# Function:     red4ext_mac::VerifyCandidateFlats(const TweakDB* db)
# Input file:   tools/probes/candidate_flats.txt (this file)
#               (read at runtime via std::ifstream; resolve path relative to
#                dyld image dir + "../../../tools/probes/candidate_flats.txt"
#                OR honor env var TWEAKXL_CANDIDATES_FILE)
# Per-line behavior:
#   - Strip comments + whitespace (drop everything from the first '#'; trim).
#   - If line non-empty: compute TweakDBID = { CRC32(name), strlen(name), {0,0,0} }
#   - Lookup(flats_map, id, GetFlatsHashMode())
#   - If hit: read FlatValue ptr at entry+0x18 via mach_vm_read_overwrite (safe);
#             read vft ptr at FlatValue+0x00; read 4 bytes at FlatValue+0x08
#             (the value); log:
#             [flat-probe] HIT name=<name> hash=0x<X> flat=<ptr> vft=0x<X> raw=0x<X>
#   - If miss:
#             [flat-probe] MISS name=<name> hash=0x<X>
#   - Once-only via std::call_once
# Wire-up:      add the call to Loader.cpp's RegisterApplyCallback after
#               VerifyFlatEntry(db).
#
# IMPORTANT for Schema — the cinema run rejected ALL 5 ExtraFlats picks. Two
# possible root causes the probe must distinguish:
#   (1) the flats genuinely aren't on those records (sparse ExtraFlats), OR
#   (2) a SYSTEMIC lookup bug (wrong CRC32 variant/seed, wrong hash mode, wrong
#       entry/value offsets). If EVERY line MISSes, suspect (2) first — sanity
#       check the TweakDBID hash against a KNOWN-present RECORD id (e.g. CRC32 of
#       "Items.Preset_Nova_Default") resolving in the RECORDS map (+0x88) before
#       trusting the flats-map (+0x58/H-008) results. Confirm flats-block offset
#       per H-008 (compare +0x58 vs +0x108 entry counts; flats >> queries).
